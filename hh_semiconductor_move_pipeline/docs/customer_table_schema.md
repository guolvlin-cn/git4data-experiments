# 华虹 MOVE 预测客户表 Schema 梳理

本文档基于 `华虹半导体MOVE预测.pdf` 中列出的客户真实表结构，并对照当前 `notebooks/pipeline` 的实际消费口径整理。目标是把“客户原始表字段”和“pipeline 派生输入/训练字段”分开说明，避免把推断指标误写成客户原字段。

## 口径说明

- PDF 中字段多为 Oracle 原始大写字段；当前 Python pipeline 读入 parquet 后通常使用小写字段名。
- 本文保留 PDF 中出现的原始拼写，包括 `WAFFER*`、`REQURED_TR`、`REQUREPDPML`、`OWNINGSTITE`、`CURRENTSTITE`、`N20EOH7`、`LASTTEPMOVE2`、`YIELDSCARP_H2` 等疑似历史拼写。
- `agg_oee_loop_day`、`agg_lotstep_loop_day`、`agg_wip_snapshot_loop_day` 是 pipeline 需要的导出/聚合结果，不是 PDF 里的原始表名。它们应由下方原始表在 Oracle 端或导出流程中生成。
- 不存在的字段不要写进客户原始 schema，例如 `bottleneck_weight`、`loop_share`、`target_move`、`actual_move`、`move_gap` 都不是客户原表字段；它们只能是后续业务输出或分析派生字段。

## 当前 pipeline 脚本范围

本目录随文档提交的 `pipeline/` 脚本来自 `notebooks/pipeline/`，但排除了 D+1 复盘归因脚本：

- 已包含：`01_load.py` 到 `09_explain.py`、`_config.py`、`run_all.py`
- 已排除：`10_metrics.py`、`11_attribution.py`

当前训练主路径消费的关键输入为：

| 输入 | 来源 | 粒度 | 当前用途 |
|---|---|---|---|
| `tb_daily_prodindex` | PDF 原始表 | product/day/productiontype 宽表 | label 源，按 MOVE 宽列 unpivot 到 `loop x op_day` |
| `agg_oee_loop_day` | `summary_oee_new` + 产能/flow 映射后的导出结果 | `loop x op_day` | OEE、down、PM、速率等外生特征 |
| `agg_lotstep_loop_day` | `tb_lotstephistory_indexes` / `rpt.lot_step_history` 聚合结果 | `loop x op_day` | 跑货活跃度、QT/RT、lot/product 数等特征 |
| `agg_wip_snapshot_loop_day` | `tb_wip_hist` + `tb_product_stage_loop` 的 08:00 快照聚合结果 | `loop x op_day` | D 日 08:00 同日已知 WIP 特征，不做 lag |
| `summary_oee_new` | PDF 原始表 | machine/day | 旧 fallback；也可用于对账和后续归因 |

```sql
FROM   mfgcim.TB_DAILY_PRODINDEX A
WHERE  A.productiontype = 'Production'
GROUP BY histdate
ORDER BY histdate DESC
```

实现补充：PDF 字段确认 `MOVE9..MOVE11` 也存在，当前 `_config.py` 已把标准 MOVE 扩到 `MOVE1..MOVE11`，再加 `BSI_MOVE1..BSI_MOVE6`、`N20_MOVE1..N20_MOVE12`，共 29 个 MOVE 宽列候选。实际运行时以现场 parquet 中存在的列与 `LOOP_COLUMN_MAP` 取交集。

## join / 聚合关系

当前可解释、可落地的关联关系如下：

| 起点 | 关联字段 | 目标 | 用途 |
|---|---|---|---|
| `tb_daily_prodindex` | `HISTDATE` + MOVE 宽列映射 | `loop x op_day` label | 训练目标 `y` |
| `tb_lotstephistory_indexes` | `PRODUCTNAME + STAGE/STAGENAME`，必要时加 step 字段 | `tb_product_stage_loop` 或 `tb_product_flow_fullstep` | 补 `LOOPNAME` / capability，再聚合到 loop-day |
| `tb_wip` / `tb_wip_hist` | `PRODUCTNAME + STAGENAME` | `tb_product_stage_loop` | WIP 快照映射到 loop |
| `summary_oee_new` | `MACHINENAME` | `tb_dsp_dspcapabilitywph` 或 capability/flow 映射 | 从 machine-day 分配/聚合到 loop-day |
| `tb_product_flow_fullstep` | `PRODUCTNAME + STAGENAME + STEP` | lot/WIP 明细 | flow、step、capability、Q-time 辅助维度 |

需要继续和客户确认的点：

- `tb_daily_prodindex` 中 `MOVE1..MOVE11`、`BSI_MOVE1..BSI_MOVE6`、`N20_MOVE1..MOVE12` 的业务 loop 名称，当前只有 `SAQP` 和 `PASSN20` 已明确。
- `summary_oee_new` 的 `PM`、`DOWN`、`WAIT_*`、`HOLD_*`、`RUN`、`AVAI` 的单位是否统一为小时，以及同一机台同一天是否能构成完整 24 小时拆分。
- `tb_dsp_dspcapabilitywph` 是否是正式可用的 WPH 基准表，`WPH`、`PEAKWPH`、`MANUALDSPWPH` 的选择优先级如何定。
- `tb_product_stage_loop` 中 `PRODUCTNAME + STAGENAME -> LOOPNAME` 是否长期唯一。
- `tb_lotstephistory_indexes.TOTALMOVE` 与 `tb_daily_prodindex` 的 MOVE 宽列是否能按日对账到同一口径。
